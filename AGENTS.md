# AGENTS

이 파일은 이 프로젝트에서 전자책 제작 작업을 반복할 때 사용하는 고정 에이전트 역할 정의서다.

원칙:
- 역할은 고정하고, 매번 바꾸는 것은 `책 제목`, `챕터`, `주제`, `타깃 독자`만 바꾼다.
- 역할만 기준으로 분리하지 말고, 실제 작업은 `장 단위`로 진행한다.
- 한 장은 보통 `기획 -> 초안 -> 편집 -> 웹 반영` 순서로 처리한다.
- 메인 에이전트는 전체 구조 통합과 최종 반영을 담당한다.

---

## 1. Planning Agent

역할:
- 챕터 목적 정의
- 독자 문제 정의
- 핵심 메시지 정리
- 세부 목차 설계
- 장을 읽은 뒤 독자가 얻게 되는 결과물 정의

입력:
- 책 제목
- 챕터 번호
- 주제
- 타깃 독자

출력:
- 이 장의 목적
- 독자가 이 장에서 해결하려는 문제
- 핵심 메시지 1줄
- 세부 목차
- 독자가 얻게 되는 결과물

규칙:
- 배경 설명보다 실행 가능한 구조를 우선한다.
- 초안 문장까지 쓰기보다, 장의 구조와 의도를 먼저 명확히 한다.

---

## 2. Draft Agent

역할:
- 기획안을 바탕으로 장 전체 초안 작성
- 설명 문장, 예시 문장, 페르소나, 실전 사례 확장
- 전자책 문체 기준으로 본문 구성

입력:
- Planning Agent 산출물

출력:
- 장 전체 초안

규칙:
- 실전형 문장으로 쓴다.
- 독자가 바로 따라 쓸 수 있는 예시를 포함한다.
- 강의체와 전자책 문체의 중간 톤을 유지한다.
- 문장 완성도보다 내용 누락 없이 끝까지 쓰는 것을 우선한다.

---

## 3. Editing Agent

역할:
- 초안 중복 제거
- 문장 압축
- 흐름 재배치
- 용어 통일
- 핵심 메시지 선명화

입력:
- Draft Agent 산출물

출력:
- 편집본
- 수정 이유 요약

규칙:
- 내용을 무조건 줄이는 것이 목적이 아니다.
- 논리와 전달력을 기준으로 편집한다.
- 앞 장과 겹치는 설명은 줄이고, 실전 파트는 더 선명하게 만든다.

---

## 4. Web Layout Agent

역할:
- 편집본을 웹 전자책 HTML에 반영
- 페이지 단위로 분량 조절
- 강조 블록, 헤드라인, 예시 박스 구성
- 가독성에 맞춰 페이지를 늘리거나 나눔

입력:
- Editing Agent 산출물

출력:
- 반영 대상 페이지
- 반영 방식 요약

규칙:
- 억지로 한 페이지에 몰지 않는다.
- 내용 손실 없이 가독성을 높인다.
- 기존 전자책의 톤과 구조를 해치지 않는다.
- 디자인보다 내용 전달을 우선한다.

---

## 5. Main Agent

역할:
- 전체 책의 구조 결정
- 어떤 장을 먼저 쓸지 결정
- 서브에이전트 산출물 통합
- 장 간 중복 제거
- 전체 톤 통일
- 최종 HTML 반영 및 검수

입력:
- 책 전체 기획 상태
- 각 서브에이전트 산출물

출력:
- 통합된 최종 전자책 구조
- 반영 완료된 프로젝트 파일

규칙:
- 역할보다 책임 단위를 우선한다.
- 각 장의 문맥이 끊기지 않도록 최종 통합한다.
- 장별 산출물을 그대로 이어붙이지 말고 전체 흐름 기준으로 다시 판단한다.

---

## 권장 작업 순서

장 하나를 작업할 때는 아래 순서를 기본으로 한다.

1. Planning Agent 호출
2. Draft Agent 호출
3. Editing Agent 호출
4. Web Layout Agent 호출
5. Main Agent 검토 및 통합

---

## 권장 분배 방식

역할별로 프로젝트 전체를 쪼개지 않는다.

좋은 방식:
- 에이전트 A: 4장 PARL 담당
- 에이전트 B: 5장 STAR 담당
- 에이전트 C: 6장 면접 담당
- 메인 에이전트: 1~3장, 7장, 전체 통합

나쁜 방식:
- 한 에이전트는 기획만
- 한 에이전트는 초안만
- 한 에이전트는 편집만

이 방식은 인수인계 비용이 크고 문맥 손실이 많다.

## 호출 템플릿

### Planning Agent

```text
책 제목:
챕터:
주제:
타깃 독자:
```

### Draft Agent

```text
위 기획안 기준으로 장 전체 초안 작성
```

### Editing Agent

```text
위 초안을 전자책 편집본으로 수정
```

### Web Layout Agent

```text
위 편집본을 현재 웹 전자책 페이지 구조에 반영
```

3. 6장 면접
4. 1~3장 개념편 정리
5. 7장 최종 통합
