■Phase 03 // 솔루션 (The Framework) PAGE_04

"초보 개발자도 즉시 활용 가능한
[정량적 자기소개서를 만드는 STAR 기법]"

어려운 기술 용어는 필요 없습니다. 인사담당자가 가장 좋아하는 건 숫자로 증명된 '성과'입니다.
누구나 해봤을 법한 '데이터 검색' 업무를 예로 들어보겠습니다.

S

Situation // 상황 설명

❌ 하수: "이전 업무 중 제품 코드를 조회했습니다."

⭕ 고수: "운영 프로세스 중 반복적인 수작업 조회가 많아 비효율이 큰 구간이 있었습니다."

T

Problem // 문제 인지 (숫자 정의)

❌ 하수: "매달 데이터를 수동으로 검색하느라 시간이 오래 걸렸습니다."

⭕ 고수: "매달 500건 이상을 수동 조회하며, 월 40시간 이상 소요되는 고비용 구조였습니다."

A

Action // 해결 방안 (작은 시도)

❌ 하수: "엑셀 단축키를 활용해 빠르게 검색했습니다."

⭕ 고수: "기준 데이터를 매핑하여 VLOOKUP으로 자동 조회되도록 시스템을 개선했습니다."

R

Result // 수치화된 성적 (결과값)

❌ 하수: "검색이 빨라져서 다른 업무를 할 여유가 생겼습니다."

⭕ 고수: "조회 시간을 80% 단축시켰으며, 월 40시간 업무를 8시간으로 최적화했습니다."

중요한 것은 결과가 얼마나 대단한지가 아닙니다. 수치가 작아도 괜찮습니다.
핵심은 내가 스스로 문제를 인지하고, 나만의 솔루션을 잡고, 해결 이후의 성과를 기록하는 '회고할 수 있는 사람'임을 보여주는 것입니다. 그것이 인사담당자가 찾는 진짜 인재의 모습입니다.

채점관의 최종 평가

"이 지원자는 단순 업무를 기술적으로 개선하여
월 32시간의 인건비를 절감할 수 있는 성과 지향적 인재임."
